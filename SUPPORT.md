# Support

This project is pre-release and currently has no guaranteed support response
time. Before requesting help, read the
[getting-started guide](docs/getting-started.md),
[configuration reference](docs/configuration.md), and
[troubleshooting guide](docs/troubleshooting.md).

Use a GitHub issue for reproducible, non-sensitive bugs and documentation
questions after the public repository is available. Include the project version,
Node version, n8n Community version, operating system, MCP client, configured
mode, safe reproduction steps, and sanitized error code. Never include secrets,
cookies, real workflow or execution payloads, or a private instance URL.

Feature requests should explain the Community Edition Public API path, user
outcome, security and compatibility implications, and why an existing tool
cannot satisfy the need.

Report vulnerabilities privately according to [SECURITY.md](SECURITY.md).
